Requirements
  • at least MyBB 1.6.0
  • PHP 5.2.0 or greater
  • MySQL as database

Description
This modification is an advanced invitations system fully integrated with MyBB. Based on a credit system and with a lot of powerful features this is the ultimate way to get more users in your forum.

Features
  • Credits
    • Give credits based on: Reputation, Posts, Days since registration, Activity on board.
    • Users can buy credits using Paypal or SMS - Fortumo.
    • Users can donate credits to other users.
    • Amount of credits to receive depending on method.
    • Max credits that regular users can have.
    • Allow users to have unlimitied credits.
  • Emails
    • Custom email subject.
    • Custom email body.
    • Commands like {INVITATION_CODE}, {REGISTRATION_URL} and much more.
    • Interpreted email body (using MyCode).
  • Keys
    • Force key, optional key or disabled keys on registration.
    • Keys are generated with an unique ID can't be copied.
    • Days while keys are valid. (Default : 7)
    • Option to add credits for expired keys.
    • Option to resend an invitation after a period of time.
  • Permissions
    • Permissions for invitations and unlimited credits.
  • Stats
    • Some stats about plugin can be found on "Forum Statistics" page.
    • Top invitees on Admin CP.
  • Others
    • Give additional credits to specific members using Admin CP.
    • Administrators can reset users number of invitations.
    • Option to show number of members invited on profile.
    • Invitations logger.
    • If a guest enter on registration process an wrong invitation code, several times, then he is banned for a while.

Installation
You can install this plugin by doing the following steps :
  1. Upload the contents of the Upload folder to the root of your MyBB installation directory.
  2. Then, install and activate Advanced Invitations System from the Administrator Panel, section Plugins.
  3. Now you can start to configure it! (Administrator Panel > Configuration > Settings > Advanced Invitations System)

Upgrade
  1. Upload the contents of the "Upload" folder to the root of your MyBB installation and overwrite all files, except system_version.php which can be found on inc/plugins/advinvsys directory.
  2. Go to Administrator Panel > Configuration > Advanced Invitations System > Modules section and click on "Run" to the next version that you want to do the upgrade.
  3. The upgrade process has done successfully.

Development
This application let you to make plugins for it. There is a class called AISDevelop, which can be found in inc/plugins/advinvsys/classes/class_develop.php file, who has a set of methods to help developers with development. Moreover, Advanced Invitations System comes with a Hello World plugin based off MyBB's one.

Some methods that you can use in order to write your own plugin for AIS :
  • addSettings($module, $settings)
    • Adds new settings
    • @param string : the codename of module which owns the setting(s)
    • @param array : the setting(s) (Include : the name (unique identifier) of the setting, the title of the setting, the description of the setting, the type of the setting ('text', 'textarea', etc...) and the value of the setting.)
    • @return bool : false if something went wrong
  • removeSettings($module, $settings = false)
    • Deletes settings from the database
    • @param string : the codename of module which owns the setting(s)
    • @param mixed : string - list of settings seperated by ',' e.g. 'test','module' , false - if you want to delete all settings from the module specified
    • @return bool : false if something went wrong
  • addLog($type, $data, $user = 1, $now = false)
    • Create a new log entry
    • @param string : type of action taken
    • @param string : extra data
    • @param int : uid of who's executed the action
    • @param bool : if it is set to "true" then the log will be insert immediatly, else will be insert later
    • @return mixed : false if something went wrong or log id if everything it is OK!
  • removeLog($ids = array())
    • Removes all log entries based on ids
    • @param array : list with log ids
    • @return bool : false if something went wrong
  • addInvitations($type, $value, $number, $reset = false, $log = true, $now = false)
    • Give or take user invites
    • @param string : user identifier column (can be 'uid' or 'username')
    • @param mixed : user identifier value (if user identifier column is 'uid' then here you must add an user id, else you need to add a string, who will represent an username)
    • @param int : the number of invitations that will be given or taken
    • @param bool : true if you would like to reset the number of invitations to the given value (user identifier value)
    • @param bool : are we going to add a log?
    • @param bool : if it is set to true then update will be maked immediatly, else after a period of time
    • @return bool : true if everything was fine or false if an error has occured
  • checkInvitations($type, $value, $amount)
    • Check if an user has a certain number of invites
    • @param string : user identifier column (can be 'uid' or 'username')
    • @param mixed : user identifier value (if user identifier column is 'uid' then here you must add an user id, else you need to add a string, who will represent an username)
    • @param int : the number of invitations that will be checked
    • @return mixed : false if it has not enough invitations or current number of invites if it has enough
  • sendPM($pm, $fromid = 0)
    • Sends a PM to a user
    • @param array: The PM to be sent; should have 'subject', 'message', 'touid' and 'receivepms' (receivepms is for admin override in case the user has disabled pm's)
    • @param int : from user id (0 if you want to use the uid of the person that sends it. -1 to use MyBB Engine
    • @return bool : true if PM was sent
and much more...
